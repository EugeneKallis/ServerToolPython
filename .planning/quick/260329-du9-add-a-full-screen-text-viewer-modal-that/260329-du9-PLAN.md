Create a new 'use client' component `TerminalOutputModal` with these props:
- `lines: string[]` — the output lines to display
- `command?: string` — optional command string to show as a header
- `onClose: () => void` — callback to close the modal

Implementation details:
- Render a fixed full-screen overlay (`fixed inset-0 z-50`) with a dark semi-transparent backdrop (`bg-black/70 backdrop-blur-sm`)
- Inside, render a near-full-screen panel (e.g., `m-4 lg:m-8` for small margins) with design system colors: `bg-surface-container-lowest border border-outline-variant`
- Panel layout: flex-col with a header bar and scrollable content area
- Header bar: shows the command (if provided) as `$ {command}` in `text-primary-fixed-dim font-mono text-xs`, plus an X close button on the right using lucide-react `X` icon, styled `text-outline hover:text-on-surface`
- Content area: `flex-1 overflow-y-auto p-4 lg:p-6` with each line rendered as a `<div>` with `whitespace-pre-wrap font-mono text-[12px] text-on-surface-variant leading-relaxed`
- Add line numbers on the left (same style as Terminal.tsx: `text-outline select-none text-[11px] w-8 shrink-0 text-right`)
- Close triggers:
  1. Click the X button in header -> calls onClose
  2. Click the backdrop (outside the panel) -> calls onClose. Use `onClick` on the outer div and `e.stopPropagation()` on the inner panel
  3. ESC key -> useEffect with keydown listener that calls onClose on Escape, cleanup on unmount
- Use `useEffect` to add `overflow-hidden` to `document.body` on mount and remove on unmount to prevent background scrolling
