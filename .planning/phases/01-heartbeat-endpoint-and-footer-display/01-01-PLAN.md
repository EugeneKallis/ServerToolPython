# Plan 01-01: Agent Heartbeat

## Objective

Add a UUID-based heartbeat coroutine to the agent worker that writes a Redis key with a 30-second TTL every 10 seconds. This allows the backend to count live agents by scanning heartbeat keys. The heartbeat must be fully isolated from command execution so failures never disrupt the worker.

## Context

The system currently has no way to know how many agent containers are running. By having each agent periodically refresh a short-lived Redis key, the backend can count active agents via key scan. The 30s TTL with 10s refresh gives a ~20s grace period before a stopped agent disappears.

<title>Add heartbeat coroutine function</title>
<read_first>
- agent/app/main.py (full file to see where to place the function — after AGENT_ID, before execute_and_stream)
</read_first>
<action>
Add the following function after the `AGENT_ID` line and before the `execute_and_stream` function:

```python
async def heartbeat(r: redis.Redis, agent_id: str):
    """Periodically refresh a Redis key so the backend can count live agents."""
    while True:
        try:
            await r.setex(f"agent:heartbeat:{agent_id}", 30, 1)
        except Exception:
            pass
        await asyncio.sleep(10)
```

Key details:
- Function signature: `async def heartbeat(r: redis.Redis, agent_id: str)`
- Uses `r.setex` with key pattern `agent:heartbeat:{agent_id}`, TTL `30`, value `1`
- Bare `except Exception: pass` — heartbeat errors must never propagate
- Sleep interval is `10` seconds
</action>
