# Plan 01-03: Frontend Agent Count Hook and Footer UI

## Objective

Create a `useAgentCount` hook that polls `GET /api/agent/count` every 15 seconds and integrate its output into the ChatTerminal footer. The footer displays a colored dot and agent count label with three visual states: loading (neutral), zero agents (red), and one-or-more agents (green).

## Context

The backend endpoint from Plan 01-02 provides the count. The frontend needs a polling hook and footer UI update. The hook returns `number | null` where `null` means "no successful fetch yet" (loading state). On fetch errors, the previous value is retained to avoid flashing incorrect counts. Design system colors come from `frontend/design/code.html`.

```typescript
import { useState, useEffect } from 'react';

export function useAgentCount(): number | null {
  const [count, setCount] = useState<number | null>(null);

  useEffect(() => {
    const fetchCount = async () => {
      try {
        const res = await fetch('/api/agent/count');
        if (res.ok) {
          const data = await res.json();
          setCount(data.count);
        }
      } catch {
        // Network error: keep previous value silently
      }
    };

    fetchCount();
    const interval = setInterval(fetchCount, 15000);
    return () => clearInterval(interval);
  }, []);

  return count;
}
```

Key details:
- Export name: `useAgentCount`
- Return type: `number | null`
- Initial state: `null` (not 0)
- Polls every `15000` ms (15 seconds)
- Fetches immediately on mount (`fetchCount()` called before `setInterval`)
- On non-ok response: does nothing (keeps previous value)
- On network error: catch block is empty (keeps previous value)
- Cleanup: `clearInterval` in useEffect return
