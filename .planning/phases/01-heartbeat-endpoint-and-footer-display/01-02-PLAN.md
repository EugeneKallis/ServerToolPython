# Plan 01-02: Backend Agent Count Endpoint

## Objective

Add a `GET /api/agent/count` endpoint to the backend that scans Redis for `agent:heartbeat:*` keys and returns a JSON count of live agents. This endpoint powers the frontend agent count display.

## Context

Once agents emit heartbeat keys (Plan 01-01), the backend needs to expose the count to the frontend. The endpoint must use `scan_iter` (not `KEYS`) to avoid blocking Redis on large keyspaces. The router is already registered in `backend/app/main.py` at prefix `/api`, so no main.py changes are needed.

## Tasks

<task id="01-02-T1">
<title>Add GET /agent/count endpoint to agent router</title>
<read_first>
- backend/app/routers/agent.py (see existing router structure, `get_redis` dependency on lines 10-15, existing `/reset` endpoint on lines 17-24)

<action>
Add the following endpoint after the existing `reset_agent` function (after line 24) in `backend/app/routers/agent.py`:

```python
@router.get("/count")
async def get_agent_count(r: redis.Redis = Depends(get_redis)):
    count = 0
    async for _ in r.scan_iter("agent:heartbeat:*"):
        count += 1
    return {"count": count}
```

Key details:
- Decorator: `@router.get("/count")`
- Function name: `get_agent_count`
- Uses the existing `get_redis` dependency (already defined in the file)
- Uses `r.scan_iter("agent:heartbeat:*")` — NOT `r.keys()` which blocks Redis
- Returns `{"count": count}` as a plain dict (FastAPI auto-serializes to JSON)
- No try/except needed — FastAPI's default 500 handler is sufficient for Redis errors on this read-only endpoint
</action>
